Repositorio:
Un repositorio es el espacio donde se almacenan los archivos de un proyecto y todo su historial de cambios. Permite organizar versiones, colaborar con otros desarrolladores y mantener trazabilidad. Es la base del control de versiones con Git.

Commit:
Un commit es un registro de cambios realizado en un momento específico del proyecto. Funciona como una “foto” del estado del código, permitiendo volver atrás si es necesario. Cada commit incluye un mensaje que explica qué se modificó y por qué.

Push:
El push es la acción de enviar los commits desde tu computador o desde Codespaces hacia GitHub. Es el paso que publica oficialmente tus cambios en el repositorio remoto para que otros puedan verlos, revisarlos o colaborar.2. conceptos-git.txt
